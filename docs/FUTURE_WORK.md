# Pablo — Future Work Backlog

*Last updated: 2026-07-03.*

This document is the single roadmap input for future planning. It reconciles two inputs:

1. **Community research** — a survey of what users of photo/DAM software actually ask for, mined from the digiKam Bugzilla wishlist (vote counts), Immich GitHub feature-request discussions (upvotes), PhotoPrism's issue tracker (reaction counts), the PhotoStructure forum, Tropy forums, XnView forums, Picasa-refugee threads and "Picasa alternative" comparisons, and Lightroom/ACDSee gap analyses.
2. **The internal defer list** — items consciously deferred during the consolidation pass, reproduced verbatim below.

Each candidate is scored on **demand × differentiation × fit with Pablo's local-first desktop model**. "Pablo today" is an honest check against what is already shipped (folder library + SQLite catalog, incremental import/rescan, face detection/recognition/naming, SigLIP2 semantic search, non-destructive editor, albums/tags/stars/captions/hide, fixed smart collections, batch rename, storage reorganizer, move/split folders, pins, file-op undo, exact+similar dedup, world map + manual geotag + KML, XMP face-region sidecars, video playback/trim, export/watermark, slideshow, print, share, collage).

Effort: **S** = days, **M** = 1–3 weeks, **L** = a month or more. Priority: **P0** = next up, **P1** = strong candidates, **P2** = worthwhile later, **P3** = park unless demand grows.

Note on vote counts: digiKam Bugzilla votes accumulate over the product's lifetime; Immich/PhotoPrism counts are from currently-open threads (checked July 2026). They are comparable within a tracker, not across trackers.

## Internal defer list (from the consolidation pass)

- ANN index for semantic/face search (linear scan fine at 30k)
- SSCD model integration / native-dedup-server merge (repository + analyzer seams ready)
- public add-on SDK (manifest loading, sandboxing, versioned ABI) — interfaces + docs shipped in docs/EXTENDING.md
- state-management rewrite + unified selection model
- core_api.dart migration onto generated ffigen bindings
- auto re-embed on face-model switch (manual 'Rebuild face index' shipped)
- background helper processes / watched folders
- Windows/Linux ONNX enablement
- .pablo-originals orphan-GC tooling

Reconciliation notes: **watched folders** appears on both lists — community demand (Picasa's most-missed trait) argues for pulling it forward despite the deferral; the P1 row above supersedes the deferral when scheduling. **SSCD integration** stays deferred (dedup already covered). The remaining defer items are engineering-health work with no direct community-demand signal and should be scheduled opportunistically alongside related feature work (e.g., ffigen migration alongside any new C API surface).

## Top-5 priorities → owning components

1. **Metadata write-back (P0)** — writer lives in `native/core/src/xmp` (XMP packet/sidecar serialization, extending the existing face-region exporter) plus `native/core/src/exif` for in-file IPTC/EXIF fields; `native/core/src/catalog` gains a dirty-metadata queue so write-back is batched and crash-safe (reuse the overwrite-with-backup path from the editor's save modes). Settings + per-field toggles surface in `pablo/lib/features/info_panel` and the organize flow in `pablo/lib/features/organize`.
2. **User-defined smart collections (P0)** — rule model, SQL compilation and persistence in `native/core/src/catalog` (the existing smart-collection queries generalize into a predicate tree exposed through `native/core/src/api` / `photo_core.h`); the semantic-clause hook calls into `native/core/src/semantic`. Rule-builder UI and collection listing live in `pablo/lib/features/sidebar` with entry points from `pablo/lib/features/search`.
3. **RAW+JPEG pairing / version stacking (P1)** — pairing detection at import/rescan time in `native/core/src/runtime` (engine + the new `analyzer.h` seam), a stack/group table in `native/core/src/catalog`, thumbnail badging via `native/core/src/thumb`; grid collapse/expand UX in `pablo/lib/features/gallery` and stack-aware operations in `pablo/lib/features/photo_tray`.
4. **OCR text search (P1)** — a new ONNX text-detection/recognition stage alongside the embedding pipeline in `native/core/src/semantic` (or as a sibling analyzer registered through `native/core/src/runtime/analyzer.h`), recognized text stored in an FTS table in `native/core/src/catalog`; blended ranking with SigLIP2 results and a "text:" filter in `pablo/lib/features/search`.
5. **Manual sort order in albums (P1)** — a position column on album membership in `native/core/src/catalog` with reorder ops exposed through `native/core/src/api`; drag-to-reorder in `pablo/lib/features/gallery` (album view) reusing the drag affordances from `pablo/lib/features/photo_tray`.
